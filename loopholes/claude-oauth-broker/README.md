# claude-oauth-broker loophole

Serializes Claude OAuth refreshes so multi-jail setups don't burn the single-use refresh token. See [`docs/claude-oauth-mitm-proxy-plan.md`](../../docs/claude-oauth-mitm-proxy-plan.md) for design rationale, [`docs/claude-token-logouts.md`](../../docs/claude-token-logouts.md) for operator triage, [`docs/loopholes.md`](../../docs/loopholes.md) for the loophole system.

## Architecture — two daemons, one loophole

This loophole ships **two** daemons wired together, neither requiring a privileged host port:

- **Host daemon** (`yolo-claude-oauth-broker-host`) — runs *on the host*, per jail, spawned by the loophole pipeline at `yolo run`. Holds the flock, reads/writes the shared credentials file, talks to Anthropic. No port binding — it listens on a Unix socket that's bind-mounted into the jail.
- **Jail daemon** (`python -m src.oauth_broker_jail`) — runs *inside the jail*, supervised at boot. Binds `127.0.0.1:443` in the container's network namespace (where :443 is unprivileged), terminates TLS for `platform.claude.com` with a CA-signed leaf cert the jail trusts, forwards refresh requests to the host daemon over the Unix socket.

Inside the jail, Claude Code opens TLS to `platform.claude.com`. `--add-host platform.claude.com:127.0.0.1` (emitted by the loophole loader) routes it to the jail daemon, which forwards to the host daemon, which holds the flock and does the real refresh.

## Files

| File | Purpose |
|---|---|
| `manifest.jsonc` | Loophole manifest with `host_daemon` + `jail_daemon` fields. Installed under `~/.local/share/yolo-jail/loopholes/claude-oauth-broker/` by `just deploy`. |
| `ca.crt`, `ca.key` | Root CA. Generated by `yolo-claude-oauth-broker-host --init-ca` at install time. Valid 10 years. |
| `server.crt`, `server.key` | Leaf cert for `platform.claude.com`, issued by the CA. Mounted into the jail read-only so the jail daemon can present it for TLS. |
| `README.md` | This file. |

## Install

`just deploy` handles everything:

1. Installs the yolo-jail wheel (exposes `yolo-claude-oauth-broker-host` on PATH).
2. Copies this directory to `~/.local/share/yolo-jail/loopholes/claude-oauth-broker/`.
3. Runs `yolo-claude-oauth-broker-host --init-ca` to generate the CA + leaf pair.
4. Retires any pre-split `claude-oauth-broker.service` systemd unit if present.

No systemd unit, no privileged port, no recurring cron. The daemons are spawned per jail.

## How requests flow

```
┌───────────────────────┐      ┌───────────────────────────┐
│  Claude Code in jail  │────▶ │  jail daemon (127.0.0.1:443)│
└───────────────────────┘ TLS  │  • terminates TLS          │
                               │  • ≠/v1/oauth/token: proxy │
                               │    to real upstream        │
                               └───────────┬───────────────┘
                                           │ Unix socket
                                           ▼
                               ┌───────────────────────────┐
                               │  host daemon              │
                               │  • flock(refresh.lock)    │
                               │  • cache check            │
                               │  • upstream refresh       │
                               │  • write shared creds     │
                               └───────────┬───────────────┘
                                           │
                                           ▼
                               Anthropic platform.claude.com
```

## Operations

```bash
# Regenerate CA/leaf (breaks existing jails until they restart and re-read the CA)
yolo-claude-oauth-broker-host --force-init-ca

# Self-check (also wired into `yolo doctor` via manifest.doctor_cmd)
yolo-claude-oauth-broker-host --self-check

# See host daemon per-jail logs
ls ~/.local/share/yolo-jail/logs/host-service-claude-oauth-broker-*.log

# See jail daemon logs (from inside a jail)
cat ~/.local/state/yolo-jail-daemons/claude-oauth-broker.log
```

## Disable

`yolo loopholes disable claude-oauth-broker` (or set `"enabled": false` in `~/.local/share/yolo-jail/loopholes/claude-oauth-broker/manifest.jsonc`), then the next `yolo run` won't spawn either daemon.

The standalone refresher (`claude-token-refresher`) remains a valid fallback — with the broker disabled, jails fall back to the single-writer refresher story and occasionally race.

## Writing your own loophole

The schema lives in [`src/loopholes.py`](../../src/loopholes.py) docstring. A loophole is a directory with `manifest.jsonc`; optional files alongside for CA, helper scripts, etc. The manifest's `host_daemon` + `jail_daemon` fields let you ship a two-sided daemon without touching yolo-jail core. See [`docs/loopholes.md`](../../docs/loopholes.md) for the full architecture and the `src/host_service.py` helper library for writing the host daemon handler.
